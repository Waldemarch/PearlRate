PearlRate

Break-even rental & mining profitability calculator dla kart NVIDIA GPU kopią Pearl (PRL) — warstwy L1 blockchain minowanej na GPU.

Pozwala w czasie rzeczywistym obliczać progi opłacalności najmu kart oraz zysk z self-miningu dla 8 architektur GPU (Blackwell, Ada, Ampere).

Cloudflare Pages KV + D1 Zero npm deps Edge Serverless Open Source
Przegląd projektu

Co rozwiązuje PearlRate?

  • 01 Niepewność cenowa najmu GPU — kalkulator oblicza maksymalny próg opłacalności be = grossH * (1 - fee) dla każdej karty, uwzględniając trudność sieci i prąd.
  • 02 Nieaktualne dane o cenie PRL — automatyczna aktualizacja co 5 minut z DexScreener, GeckoTerminal i CoinGecko przez Worker Cron.
  • 03 Brak porównania GPU — tabela dla 8 kart (RTX 5090 → RTX 3060) z edytowalnymi wydajnościami yields i ilościami qty.
  • 04 Historyczny trend cen — wykres SVG z pobieraniem historii przez endpoint /api/history?range=7d z D1.

Request Lifecycle — Krok po Kroku

🔄 Przepływ aktualizacji ceny
  • 1Worker Cron wyzwalany co */5 * * * * przez Cloudflare Triggers
  • 2Próba fetch z DexScreener (token: 0x07696d...cb00 na ethereum)
  • 3Fallback → GeckoTerminal (network: "eth"), fallback → CoinGecko ("wrapped-pearl")
  • 4Zapis do KV (klucz "prl_price") i D1 INSERT (prl_price)
🌐 Przepływ żądania użytkownika
  • 1Browser ładuje index.html z CDN (Cloudflare Pages)
  • 2JS wykonuje GET /api/price z timeoutem 6 sekund (AbortController)
  • 3Pages Function czyta KV → zwraca {"price","ts","source"}, nagłówek Cache-Control: no-store
  • 4JS wywołuje applyLivePrice(p) → aktualizuje S.price i przebudowuje UI przez compute()
  • 5Wykres: GET /api/history?range=7d → D1 query → downsampling → SVG drawChart()

Podsumowanie

Dlaczego ten stack działa dla PearlRate?

⚡ Zero Cold Start
Cloudflare Workers startują w <1 ms na edge. KV reads są globalnie dystrybuowane — brak latencji DB na najpopularniejszy endpoint GET /api/price.
💰 Ekstremalnie niski koszt
Zero npm, zero bundlera = zero czasu budowania i zero kosztów serwerowych. Cloudflare Pages free tier obsługuje produkcję. D1 i KV mają hojne free tiers.
🔄 Self-updating
Cron Worker (*/5 * * * *) z 3-poziomowym fallbackiem (DexScreener → GeckoTerminal → CoinGecko) zapewnia dostępność cen bez ręcznej interwencji.
🗄️ Odpowiedni storage
KV dla atomic "latest price" (szybkość), D1 SQLite dla historycznych serii (analityka). Każde rozwiązanie dopasowane do swojego use-case.
📱 Offline-capable
localStorage (pearlrate:state) persystuje cały stan kalkulatora. Aplikacja działa w pełni bez internetu — API to enhancement, nie requirement.
🚀 Developer Experience
Jeden plik index.html, brak konfiguracji, deploy przez git push. Nowy deweloper ma działający serwis w minuty przez wrangler pages dev ..
